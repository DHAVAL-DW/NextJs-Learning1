⚠️ Common PHP → Next.js Mistakes & Prevention

💡 Why This Matters

PHP developers often carry over familiar patterns that become anti-patterns in Next.js. Understanding these common pitfalls early prevents technical debt and security vulnerabilities in production applications.

PHP Habit Next.js Anti-Pattern Correct Approach Prevention
🔄 Mixing logic and presentation Putting data fetching in Client Components Use Server Components for data, Client Components for interactivity
Server Components render on the server, reducing JS bundle size and improving performance [^15^] 		Code review checklist
🍪 Session-based state Storing user state in localStorage Use HTTP-only cookies + Server Components
localStorage is vulnerable to XSS attacks; HTTP-only cookies are secure and work with Server Components 		Security audit Week 3
⏳ Synchronous thinking Blocking renders with heavy computations Use streaming, Suspense boundaries
React Suspense allows components to "wait" for something before rendering, improving perceived performance 		Performance budget
🌍 Global variables Using window object without checks Proper SSR safety checks, use hooks
Always check typeof window !== 'undefined' or use useEffect for browser-only APIs 		ESLint rules
🐛 Echo/print debugging Console.log in production Structured logging, error boundaries
Use services like Sentry or LogRocket for production monitoring; implement React Error Boundaries 		Code review
📦 Monolithic controllers Giant API route handlers Colocated Server Actions, smaller functions
Server Actions allow form mutations without API routes; keep functions focused and single-purpose [^8^] 		Architecture review
🗄️ Direct SQL queries Raw queries without ORM Prisma with type safety
Prisma provides type-safe database access, auto-generated migrations, and connection pooling 		Mandatory ORM usage
📁 File uploads to local disk Storing uploads in /public Cloud storage (S3, Cloudinary)
Serverless environments don't persist local files; use object storage with CDN for static assets 		Infrastructure template
🔧 Ignoring build process Developing without `next build` Build in CI on every PR
Static generation issues only appear during build; catch them early with CI/CD pipelines 		CI/CD enforcement
🛣️ PHP-style routing Manual URL parsing File-based App Router
Next.js App Router uses file-system based routing; no manual route configuration needed [^8^] 		Routing workshop Day 6
